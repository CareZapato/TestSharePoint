Laboratorio de Integración SharePoint - Sistema de Pagos de Seguros

Tecnologías: Spring Boot + Microsoft Graph API + CSV

📖 Guía de Uso - SharePoint Payment Lab

🚀 Inicio Rápido

  1. Haz clic en "Cargar Datos Mock" para cargar datos de ejemplo
  2. Explora los pagos en la tabla
  3. Prueba los diferentes escenarios de simulación
  4. Genera reportes CSV y envíalos a SharePoint

🎮 Panel de Control

  • 🔄 Cargar Datos Mock: Carga los datos de ejemplo desde el archivo JSON
  • 📄 Generar CSV: Crea un archivo CSV con los pagos actuales y lo guarda localmente
  • ☁️ Enviar a SharePoint: Genera el CSV y lo sube a SharePoint (si está configurado) o lo guarda localmente
  • ⚙️ Estado SharePoint: Verifica si SharePoint está configurado correctamente

🎭 Escenarios de Simulación

  • 💳 Pago en Línea Confirmado: Simula un pago online exitoso e imputado
  • 🔁 Pago Automático Confirmado: Simula un pago recurrente procesado correctamente
  • ❌ Pago Rechazado: Simula un pago rechazado por fondos insuficientes
  • 📦 Lote de Confirmados: Genera 5 pagos confirmados para reportes
  • ⏳ Cuotas Pendientes: Genera cuotas sin pagar para un RUT específico

🔍 Búsqueda por RUT

Ingresa un RUT en el campo de búsqueda para ver todos los pagos de ese cliente.

RUTs de ejemplo:

  • 12345678-9 - Pago confirmado
  • 15555666-7 - Cuotas pendientes
  • 11223344-5 - Pago rechazado

☁️ Configuración de SharePoint

Modo Local (por defecto):

  • No requiere configuración
  • Los archivos CSV se guardan en ./generated-reports/

Modo SharePoint:

  • Configura las variables de entorno (ver archivo .env.example)
  • Necesitas: Tenant ID, Client ID, Client Secret, Site ID, Drive ID
  • Consulta el README.md para instrucciones detalladas

📊 Formato del Reporte CSV

Los archivos generados incluyen:

  • RUT, Email, Póliza, Prima, Cuota
  • Monto, Canal, Pasarela, Estado
  • Imputado, Fecha Pago, Fecha Imputación
  • Descripción

Nombre del archivo: reporte-pagos-YYYYMMDD-HHmmss.csv

💡 Consejos

  • Siempre verifica el estado de SharePoint antes de enviar archivos
  • Los datos mock se resetean al aplicar un nuevo escenario
  • Puedes usar "Cargar Datos Mock" para volver a los datos iniciales
  • Los mensajes en la parte superior te guiarán en cada acción
